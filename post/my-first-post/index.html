How To Blog On Github From Zero With Hugo And Github Pages

October 6, 2024

This blog is suitable for someone who want to start a blog on github from zero to start with hugo and github pages. The blog contains two parts which are steps and some problems that you probably encounter. You can share some simple ideas about technology, life, etc on blog. If you have high demand for security and privacy, you can read other blogs to meet your needs.

part 1:steps(I use vscode to run)

step 1: create a new repository

1.If you don’t have a github account, you can sign in on Github.

2.Create a new repository named username.github.io. The username is your github account name.

step 2: install hugo and git

1.Install hugo on your computer. You can download the package from Hugo Releases. More details can be found on Hugo Docs.(For me, I use windows 11.I just need to download the package and then comfigure the environment variable.)

In terminal, use hugo version to make sure if the installation is successful.

hugo version

2.Install git on your computer. You can download the package from Git Download.

In terminal, use git version to make sure if the installation is successful.

git version

step 3: create a new site

1.enter your own folder.(The folder is to store your local repository.)

2.creat a new hugo site(You can use the name you like to replace DelongYang666-blog)

hugo new site DelongYang666-blog

step 4: add a theme

1.choose a theme from Hugo Themes

2.download the theme.Then you can see the “ananke” in the theme folder.

cd DelongYang666-blog

git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

3.configure the theme in hugo.toml

cd DelongYang666-blog

add theme = ‘ananke’ to the hugo.toml in the end by hand.

step 5: creat a new blog 

cd DelongYang666-blog

hugo new post/my-first-post.md

edit the post/my-first-post.md and save it.

step 6:preview site

cd DelongYang666-blog

hugo server -D

enter the url in the browser. The url is http://localhost:1313/. You can see your first blog.Press ctrl + c to exit the server.

step 7:creat a hugo site

cd DelongYang666-blog

hugo

then you can see some documents in the public folder.The public folder is the folder to store the site.

step 8:convert the public folder to git library

cd DelongYang666-blog/public

git init

step 9:associate a local library with a remote library 

cd DelongYang666-blog/public

git remote add origin git@github.com:DelongYang666/DelongYang666.github.io.git

change DelongYang666/DelongYang666.github.io.git to your own name.

open public folder ->.git -> config document. You can see if the remote is associated with the local library.

	repositoryformatversion = 0
	filemode = false
	bare = false
	logallrefupdates = true
	symlinks = false
	ignorecase = true
[remote "origin"]
	url =git@github.com:DelongYang666/DelongYang666.github.io.git
	fetch = +refs/heads/*:refs/remotes/origin/*

step 10:commit changes to the local library 

cd DelongYang666-blog/public

git status#see current status

git add .#add all files you chaged

git commit -m "Add a new post"#commit changes

step 11:push changes to the remote library 

cd DelongYang666-blog/public

git push -u origin master

part 2:problems that you probably encounter

problem 1. *** Please tell me who you are. Run git config –global user.email “you@example.com” git config –global user.name “Your Name” 

git config --global user.email "you@example.com"

git config --global user.name "Your Name"

problem 2. git@github.com: Permission denied (publickey). fatal: Could not read from remote repository. Please make sure you have the correct access rights and the repository exists. 

ssh-keygen -t rsa -C "youremail@example.com"

chage the email to your own email.Press enter all the time.

enter C:\User\root.ssh.Open the id_rsa.pub document.Copy the content.

Open your github.Press setting -> SSH and GPG keys.Fill in the ‘Ttile’ blank with ‘key’.Fill in the ‘Key’ blank with the content you copied.

then add the ssh key.

problem 3. ssh: connect to host github.com port 22: Connection refused fatal: Could not read from remote repository. Please make sure you have the correct access rights and the repository exists. 

cd C:\Users\username\.ssh

then you can see the config document.Open it and add the following content.

Host github.com
 Hostname ssh.github.com
 Port 443

problem 4. ! [reject!] master -> master (fetch first)

error:failed to push some prefs to ‘xxxxxx’

first way(suit for the first push)

git push -f origin master

second way(suit for the daily push)

git pull origin master --allow-unrelated-histories #Synchronize the remote repository with the local repository
git add .
git commit -m "commit messege"
git push -u origin master

problem 5. src refspec master does not match any error: failed to push some refs to

The default branch on the github is ‘main’,if you press ‘git push -u origin master’.It will be error.

first way

git push -u origin main

second way

open your github and enter your repository.press settings -> page ->source ->deploy from a branch.branch->master.remember to press save.
